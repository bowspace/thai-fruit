# ThaiFruit ‚Äî Design Guideline

> This file is read by the `impeccable` skill and by any agent working on frontend features. It is the source of truth for **how ThaiFruit should look and feel**. Update it when the aesthetic direction or tokens change.

---

## Design Context

### Users

- **Buyers** ‚Äî Thai/English/Chinese-speaking end consumers shopping for fresh Thai fruit. Primarily on **mobile** (evenings, one-hand, sometimes in a market or on transit). Want to browse, compare, trust the seller, and checkout quickly.
- **Sellers** ‚Äî Farmers/store owners managing products, stock, and orders. Primarily on **desktop** (morning ops, during packing). Want fast table/form workflows ‚Äî no-nonsense dashboard, not a "fancy" admin.
- **Job-to-be-done** ‚Äî Buyers: "find good fruit from someone I trust and get it to me fresh." Sellers: "run my store without the tool getting in the way."

### Brand Personality

Three words: **fresh ¬∑ trustworthy ¬∑ modern-craft**.

- Fresh ‚Äî the product is alive (ripe fruit, farm air). The UI should breathe, not feel plastic.
- Trustworthy ‚Äî direct-from-farmer is the differentiator. Typography, spacing, and photography should feel considered, not mass-market.
- Modern-craft ‚Äî clean like Airbnb/Grab, curated like Whole Foods / a farmers market. Not a discount megastore.

**Emotional goal:** the user should feel like they just walked into a well-kept neighborhood market ‚Äî organized, generous, honest pricing, no shouting.

### Aesthetic Direction

- **Theme:** Light mode. Warm, high-contrast, photo-forward.
- **References (do feel like):**
  - Whole Foods / local farmers market ‚Äî curated abundance, natural palette, confidence in the product.
  - Airbnb, Grab, Gojek consumer surfaces ‚Äî rounded, generous whitespace, photography doing the heavy lifting.
- **Anti-references (do NOT feel like):**
  - Shopee / Lazada / AliExpress ‚Äî dense red/orange discount stickers, banner clutter, countdown timers, confetti pricing.
  - Generic "admin dashboard with colored stripe on every card."
  - Hospital / enterprise SaaS ‚Äî sterile, cold, gray.

### Design Principles

1. **Product first, chrome second.** Photography of fruit is the hero. Reduce borders, stickers, and UI scaffolding around it.
2. **Calm hierarchy.** One thing is loud per screen ‚Äî usually the price or the primary CTA. Everything else defers.
3. **Warm neutrals, not gray.** Backgrounds are tinted toward the brand green. Pure gray reads as clinical.
4. **Generous rhythm.** Spacing does hierarchy work. Don't add borders where whitespace is enough.
5. **Buyer is mobile, seller is desktop** ‚Äî but both share tokens. Layouts adapt; tokens don't fork.
6. **Thai is the primary voice.** Thai copy sets line-height and font-weight expectations; EN/CN adapt.

## Design Tokens

All tokens live in `src/index.css` under `:root`. **Use the CSS variable, never the raw value.** If a value doesn't have a token yet and you'd use it more than twice, add a token.

**Primary ‚Äî lush green (brand identity):**
- `--primary: #1a6b3c` ‚Äî primary brand, buttons, links, emphasis
- `--primary-light: #2d9254` ‚Äî hover states, gradient endpoints
- `--primary-lighter: #e8f5ee` ‚Äî subtle backgrounds, selected chips, store avatars
- `--primary-dark: #0d4a28` ‚Äî headers, footer, deep accents

**Accent ‚Äî warm golden (calls-to-action, highlights):**
- `--accent: #f59e0b` ‚Äî badges, cart CTA, "premium" markers
- `--accent-light: #fbbf24`
- `--accent-dark: #d97706` ‚Äî accent hover

**Secondary ‚Äî orange (pricing):**
- `--orange: #ea580c` ‚Äî **ONLY for prices** (reserved semantic). Don't use for generic UI.
- `--orange-light: #fb923c`

**Neutrals (tinted green ‚Äî never pure gray):**
- `--bg: #f8faf9` ‚Äî page background
- `--bg-card: #ffffff` ‚Äî card surfaces
- `--bg-hover: #f1f8f4` ‚Äî row hover
- `--text: #1a2e1a` ‚Äî body text
- `--text-secondary: #4b6b4b` ‚Äî supporting copy
- `--text-muted: #8ca38c` ‚Äî meta, timestamps, hints
- `--border: #d4e8d8` ‚Äî default borders
- `--border-light: #e8f0ea` ‚Äî subtle dividers
- `--divider: #eef4f0` ‚Äî inside-card separators

**Status (order states):**
- `--status-pending` / `--status-pending-text` ‚Äî amber
- `--status-confirmed` / `--status-confirmed-text` ‚Äî green
- `--status-shipped` / `--status-shipped-text` ‚Äî blue
- `--status-done` / `--status-done-text` ‚Äî green

**Color rules:**
- Prices are always `--orange`. Buttons/CTAs are `--primary` or `--accent` gradient.
- Text on green backgrounds: white or `--accent`, never gray.
- Don't introduce new hues without adding a token.

### Radius

- `--radius-sm: 8px` ‚Äî small controls, thumbs, inputs-inside-editor
- `--radius: 12px` ‚Äî default (inputs, buttons, chips)
- `--radius-lg: 16px` ‚Äî cards, table wrappers
- `--radius-xl: 24px` ‚Äî hero, modal shell, product detail hero
- `--radius-full: 9999px` ‚Äî pills, badges, avatars

### Shadow

- `--shadow-sm` ‚Äî resting card lift
- `--shadow` ‚Äî hover lift
- `--shadow-md` ‚Äî emphasized hover / dropdown
- `--shadow-lg` ‚Äî modal / product card hover
- `--shadow-glow` ‚Äî focus ring for primary surfaces

All shadows are **green-tinted** (`rgba(26,107,60, ‚Ä¶)`), not black ‚Äî this is what keeps the UI warm.

### Motion

- `--ease: cubic-bezier(0.4, 0, 0.2, 1)` ‚Äî default (material standard)
- `--spring: cubic-bezier(0.34, 1.56, 0.64, 1)` ‚Äî playful emphasis (toast, modal slide-up, cart badge pop)

**Durations:** 0.15s (hover), 0.2s‚Äì0.3s (state change), 0.4s (entrance). Don't invent new durations.

**Keyframes available:** `fadeIn`, `slideUp`, `popIn`, `toastSlideIn` ‚Äî reuse rather than redefining.

### Spacing ‚Äî 4pt scale tokens

`--space-1` (4px) ¬∑ `--space-2` (8px) ¬∑ `--space-3` (12px) ¬∑ `--space-4` (16px) ¬∑ `--space-5` (20px) ¬∑ `--space-6` (24px) ¬∑ `--space-8` (32px) ¬∑ `--space-10` (40px) ¬∑ `--space-12` (48px) ¬∑ `--space-16` (64px). Prefer these over raw px in new code. Existing components still use raw px ‚Äî fine to leave; convert when touching.

### Typography

**Two stacks, exposed as tokens** (`index.html` loads from Google Fonts):
- `--font-body: 'Inter', 'Noto Sans Thai', system-ui, sans-serif` ‚Äî body, controls, micro
- `--font-display: 'Fraunces', 'Noto Serif Thai', Georgia, serif` ‚Äî hero/page H1s, section titles, prices on detail pages, big stat numbers

**Use display where it has earned its weight.** Headlines, store/product names, prices in detail/cart, stat values, and editorial moments (`.season-card-num`, `.season-card-fruit`). Body copy, controls, and meta stay on `--font-body`.

**Scale used in the app** (clamps for headlines, fixed for body):
- Hero H1 ‚Äî `clamp(34px, 5vw, 56px) / 600` Fraunces
- Page H1 ‚Äî `clamp(32px, 4vw, 48px) / 600` Fraunces (product-name, store cover)
- Display section title (`.section-title--display`) ‚Äî `clamp(22px, 2.6vw, 30px) / 600` Fraunces
- Section title (default) ‚Äî 22px / 800 Inter
- Display price (`.pd-price-display`) ‚Äî `clamp(48px, 6vw, 72px) / 600` Fraunces, color `--orange`
- Card title ‚Äî 17px / 700 (store), 15px / 700 (product) Inter
- Body ‚Äî 14‚Äì16px / 400 Inter
- Meta ‚Äî 12‚Äì13px / 400‚Äì500 Inter
- Micro ‚Äî 11px / 600 (badges, labels) Inter

**Typography rules:**
- Display face is for moments, not paragraphs. Don't put body copy in Fraunces ‚Äî Inter handles all running text and Thai/CN.
- Use font-weight for hierarchy, not size-alone. Display 600 reads heavier than Inter 800.
- Thai copy needs `line-height: 1.6`+ ‚Äî don't tighten below that.
- Italic Fraunces is reserved for the accent word in hero H1, the editorial numeral in `.season-card-num`, and the season "fruit" labels.
- Tabular numerics: stat values and money columns use `font-variant-numeric: tabular-nums` for clean alignment.
- Uppercase is reserved for `modal-section-label`, micro-badges, and uppercase meta labels (`.cart-store-pickup`, `.season-card-meta`, `.seller-header-meta`). Don't uppercase body copy.
- Letter-spacing `-0.02em` for hero/cover Fraunces; `-0.01em` for section/card display titles. Body unchanged.

## Patterns

### Page shell
`.main` wraps page content with `max-width: 1280px`, `padding: 24px` (16px mobile). Every top-level page should sit inside it.

### Section with rule
```jsx
<h2 className="section-title">
  <span className="section-title-icon">üçä</span>
  Section name
</h2>
```
The trailing gradient rule is built into `::after`. Don't add a custom underline.

### Empty state
Always use `.empty` with `empty-icon`, `h3`, and `p`. Never leave a blank list ‚Äî teach what to do next.

### CTA hierarchy per screen
One primary (`.btn-primary` / `.btn-add-cart` / `.btn-checkout`). Everything else is `.btn-secondary` or a ghost/text link. If a screen needs two primaries, one of them is wrong.

### i18n hygiene
- User-facing strings: `t('key')` (add to `src/context/LangContext.jsx`).
- Data fields: `locField(obj, 'name')` (reads `name` / `nameEn` / `nameCn`).
- Never hardcode Thai/EN/CN strings in JSX.

## Do / Don't for ThaiFruit

### Do
- Let photography of fruit carry the page. Default to larger, more generous product images on mobile.
- Use `--primary-lighter` as a tint for "grouped" zones (cart header, store avatar background, category badges) ‚Äî it signals "this belongs together" without a border.
- Reuse the existing keyframes (`fadeIn`, `slideUp`, `popIn`, `toastSlideIn`). They already match the `--spring` easing.
- Use `gap` over margins for sibling spacing (already the convention).
- When adding a new page, add its CSS block under a `/* ==== PAGE NAME ==== */` banner in `src/index.css` and prefix classes (`.pd-*`, `.cart-*`) ‚Äî don't create new CSS files.

### Don't
- **Don't copy Shopee/Lazada patterns:** no red discount stickers plastered on every card, no countdown timers, no confetti "deal" banners, no "1,234 sold" social-proof tags, no flash-sale sidebars.
- **Don't add colored left/right border stripes (>1px) on cards, alerts, or list rows.** The top-of-card `::before` gradient on `.stat-card` and `.store-card` is fine ‚Äî it's already the house pattern. A new left stripe is not.
- **Don't use gradient text** (`background-clip: text`). Solid color for all text. If you want emphasis, use weight or size.
- **Don't use pure black or pure white for UI surfaces.** `--text` and `--bg-card` are already tinted; use the tokens.
- **Don't reach for a modal by default.** Navigate to a page (existing `page`-state routing) or expand inline. `ProductModal.jsx` was deprecated in favor of `ProductDetail.jsx` for this reason.
- **Don't use gray text on colored backgrounds.** On green headers, text should be white or `--accent`.
- **Don't add per-feature ad-hoc spacing values.** Use the radius / shadow / ease tokens. If a new value is truly needed, add a token first.
- **Don't introduce Material/Chakra/shadcn components.** This project is hand-rolled CSS + lucide-react icons. Match the style.
- **Don't hide critical functionality on mobile** (current search-bar `display: none` under 768px is a known gap, not a pattern to copy). Adapt ‚Äî don't amputate.
- **Don't center-align long-form layout.** Left-align is the house voice. Center only for `.empty`, logo/hero, login modal.

## Accessibility Baseline

- Keep contrast ‚â• 4.5:1 for body text on any background. The current `--text-secondary` on `--bg-card` meets this; don't go lighter.
- Inputs use a 3px `rgba(26,107,60,0.1)` focus ring ‚Äî preserve on any new form control.
- Touch targets ‚â• 44px on mobile (qty buttons already use 44√ó44). Don't go smaller.
- Avoid removing focus outlines without providing a visible replacement.
- Preserve `html lang="th"` and per-element `lang` attributes when rendering CN/EN fallbacks inline.

## When extending the design system

Before adding anything new, in order:

1. **Reuse an existing class** from the inventory above.
2. **Extend with a variant** (e.g., `.btn-primary.btn-small`) if the component is 80%+ the same.
3. **Add a new component only if it's used in ‚â•3 places with the same intent.** Premature abstraction is worse than duplication here.
4. **Add a CSS block** under `src/index.css` with a `/* ==== NAME ==== */` banner, not a new file.
5. **Add tokens** if you find yourself typing a hex/px value more than twice.
6. **Update this file** when the pattern solidifies.

## Known tensions & future premium-ification

The baseline is in place but the user has asked for a more **premium / Whole Foods / Airbnb** tone. These are candidate upgrades ‚Äî do **not** do them in a one-off feature PR. Propose them as a dedicated design pass:

- ‚úÖ **Typography** ‚Äî *Done in `redesign/all-pages` branch.* Fraunces (Latin) + Noto Serif Thai (Thai display) loaded; `--font-display` token live; applied to hero/H1/section-title-display/PDP price/seller stat numbers/season strip.
- ‚úÖ **Spacing rhythm** ‚Äî *Done.* `--space-1` ‚Ä¶ `--space-16` 4pt-scale tokens added; new code consumes them. Older blocks still use raw px and convert opportunistically.
- **Color space** ‚Äî migrate tokens to `oklch()` so hover/active states can be derived with `color-mix()` instead of hand-picked. Perceptually uniform, fewer magic values.
- **Photography** ‚Äî product images are currently placeholder-heavy. Real farm photography is the single biggest premium lever ‚Äî more than any code change. Note: image slots are now 4:5 portrait everywhere (`.product-img` aspect-ratio change), so the moment real photos arrive the design transforms automatically.
- **Mobile header search** ‚Äî still hidden under 768px with no fallback. A mobile search sheet remains a candidate for a follow-up.
- **Cart qty stepper** ‚Äî buyers currently change qty by removing and re-adding. Adding inline `.qty-ctrl` to cart rows would need a new `updateCartItemQty` action in `AppContext`.

These are notes for later, not TODOs for today.
